Open Data API · v1

API de Dados do Ijuí City Lab

Acesse de forma programática os dados publicados pelos projetos do sandbox e envie automaticamente os dados das suas soluções. Base aberta para você usar em pesquisas, dashboards e seus próprios projetos.

1. Autenticação

Toda requisição exige uma chave de API no cabeçalho X-API-Key. Qualquer participante cadastrado pode gerar chaves gratuitas no portal (aba API & Dados). Cada chave tem um escopo: read (ler), ingest (enviar) ou read_ingest (ambos).

X-API-Key: icl_xxxxxxxxxxxx_xxxxxxxxxxxxxxxxxxxxxxxx

A chave completa é exibida uma única vez, no momento da criação. Guarde-a com segurança; se vazar, revogue e gere outra.

2. Listar projetos publicados

GET /api/v1/projects

curl https://SEU_DOMINIO/api/v1/projects \
  -H "X-API-Key: SUA_CHAVE"

Resposta:

{
  "count": 2,
  "projects": [
    {
      "slug": "monitoramento-trafego-ia",

}

3. Ler medições (dados abertos)

GET /api/v1/measurements

ParâmetroDescrição
projectObrigatório. Slug do projeto.
metricFiltra por uma métrica específica.
from / toIntervalo (ISO 8601) sobre recorded_at.
limitMáximo de registros (padrão 1000, máx 10000).
formatjson (padrão) ou csv.
# JSON
curl "https://SEU_DOMINIO/api/v1/measurements?project=monitoramento-trafego-ia&metric=veiculos_hora&limit=100" \
  -H "X-API-Key: SUA_CHAVE"

# CSV (para planilhas / análise)
curl "https://SEU_DOMINIO/api/v1/measurements?project=monitoramento-trafego-ia&format=csv" \
  -H "X-API-Key: SUA_CHAVE" -o dados.csv
{
  "project": "monitoramento-trafego-ia",
  "count": 2,
  "measurements": [
    { "metric": "veiculos_hora", "value": 412, "unit": "veic/h", "recorded_at": "2026-06-15 18:00:00" },
    { "metric": "veiculos_hora", "value": 388, "unit": "veic/h", "recorded_at": "2026-06-15 17:00:00" }
  ]
}

4. Enviar dados (ingestão)

POST /api/v1/ingest — requer chave com escopo ingest ou read_ingest, pertencente ao dono do projeto.

Uma medição:

curl -X POST https://SEU_DOMINIO/api/v1/ingest \
  -H "X-API-Key: SUA_CHAVE" \
  -H "Content-Type: application/json" \
  -d '{"project":"meu-projeto","metric":"temperatura","value":23.4,"unit":"C"}'

Várias de uma vez (até 1000):

curl -X POST https://SEU_DOMINIO/api/v1/ingest \
  -H "X-API-Key: SUA_CHAVE" \
  -H "Content-Type: application/json" \
  -d '{
        "project": "meu-projeto",
        "measurements": [
          { "metric": "pm25", "value": 12.1, "unit": "ug/m3", "recorded_at": "2026-06-15T18:00:00Z" },
          { "metric": "ruido", "value": 58.3, "unit": "dB" }
        ]
      }'

Os dados enviados aparecem no painel ao vivo da home (se o projeto estiver publicado) e ficam disponíveis na API de leitura para terceiros.
